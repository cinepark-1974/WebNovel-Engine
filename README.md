## 3단계 파이프라인

```
STEP 1. CONCEPT (아이디어 / 기획서 / 직접입력)
   ├── 기획서 업로드 (DOCX/HWP/PDF/TXT 자동 파싱)
   ├── 한 줄 아이디어 → 컨셉 카드 자동 생성
   ├── 9칸 직접 입력 + 서사 모티프 선택 + 반동인물 주인공 체크
   ├── 자동 보강 (빌런 / Fatal Flaw / 미스터리 축)
   └── 연재 설정 (플랫폼 · Core/Extension · 수위 · 과금)
          ↓
STEP 2. BUILD-UP
   ├── 2-1. Core Arc (25화 × 2청크, 50화 완결형)
   ├── 2-2. 떡밥 맵 (Plant & Payoff + Extension 예비)
   ├── 2-3. 캐릭터 바이블
   ├── 2-4. 회차 플롯 (🚀 일괄 생성 / 📋 개별 재생성 모드)
   └── 2-5. 🆙 Extension Arc (인기 대응 확장)
          ↓
STEP 3. WRITING
   ├── 3-1. 19금 원고 집필 (Opus, 플랫폼별 분량 + 모티프 적용)
   ├── 3-2. 15금 변환 (Sonnet)
   ├── 3-3. 품질 체크
   └── 3-4. 내보내기 (TXT + DOCX + 기획서 DOCX)

💾 전역: 프로젝트 저장/불러오기 (세션 타임아웃 대응)
```

## 서사 모티프 이중 구조 (신규)

고경은(2022) 연구 기반 — 웹소설의 핵심 서사 문법을 1차 + 2차로 분리:

**1차 모티프 (시공간 이동) — 6종:**
회귀 · 빙의 · 환생 · 귀환 · 차원이동 · 일상

**2차 모티프 (소재 결합) — 12종:**
성장물 · 먼치킨물 · 사이다물 · 시한부물 · 책빙의물 · 육아물 · 법정물 · 학원물 · 아이돌물 · 전문직물 · 연예계물 · 헌터물

```
예: "회귀" + "사이다물" = 회귀해서 억울함 청산
예: "환생" + "육아물" = 환생해서 딸 키우기
예: "빙의" + "시스템물" = 소설 속 시스템 플레이어
```

- 관능 씬의 신체 반응·감정 변화는 **여자 주인공 시점**

**관능 씬의 4대 서사 기능** (1개 이상 포함):
1. 성적 만족 제공
2. 부성 권위 대체
3. 경제적/사회적 안정 상징
4. 정서적 성장 보조

## Core Arc + Extension 구조

```
[Core Arc 50화]        독립 완결 가능
  ↓ 반응 지표 확인
[Extension +25~50화]   인기 대응 확장 (3모드)
  ↓
[Season 2]             장기 IP화
```

**Extension 3모드:** Bridge (중반 30~35화) / Late (후반 45화) / Season Break (완결 후)

전역 💾 프로젝트 저장 버튼 — 세션 전체를 JSON으로 백업/복원:
