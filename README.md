## 소개

Web Novel Engine v2.1은 한국 웹소설 플랫폼(리디, 카카오페이지, 문피아, 네이버 등)의 실전 양식에 맞춰 설계된 AI 콘텐츠 생산 엔진입니다. 사전제작의 안정성과 연재 반응 대응의 유연성을 결합한 **Core Arc + Extension** 구조, 플랫폼별 분량 자동 적용, 수익에 직결되는 **회차 제목 엔진**까지 포함합니다.

---

## 3단계 파이프라인

```
STEP 1. CONCEPT (아이디어 / 기획서 / 직접입력)
   ├── 기획서 업로드 (DOCX/HWP/PDF/TXT 자동 파싱)
   ├── 한 줄 아이디어 → 컨셉 카드 자동 생성
   ├── 9칸 직접 입력
   ├── 자동 보강 (빌런 / Fatal Flaw / 미스터리 축)
   └── 연재 설정 (플랫폼 · Core/Extension · 수위 · 과금)
          ↓
STEP 2. BUILD-UP
   ├── 2-1. Core Arc (25화 × 2청크, 50화 완결형)
   ├── 2-2. 떡밥 맵 (Plant & Payoff + Extension 예비)
   ├── 2-3. 캐릭터 바이블
   ├── 2-4. 회차 플롯 (도입/전개/훅 + 제목 자동 생성)
   └── 2-5. 🆙 Extension Arc (인기 대응 확장)
          ↓
STEP 3. WRITING
   ├── 3-1. 19금 원고 집필 (Opus, 플랫폼별 분량)
   ├── 3-2. 15금 변환 (Sonnet)
   ├── 3-3. 품질 체크
   └── 3-4. 내보내기 (TXT + DOCX + 기획서 DOCX)
```

## 플랫폼별 분량 자동 적용

| 플랫폼 | 1화 분량 (공백 포함) | 비고 |
|--------|---------------------|------|
| 카카오페이지 | 5,000~5,500자 | 표준 |
| 리디북스 | 5,000~5,500자 | 표준 |
| 문피아 | 5,000~5,500자 | 5,000자 미달 시 유료 불가 |
| 네이버시리즈 | 5,500~6,000자 | 가장 길다 |
| 네이버웹소설 | 6,000~6,500자 | 정식 연재·공모전 기준 |
| 조아라 | 4,000~4,500자 | 무료연재 |

STEP 1에서 플랫폼 선택 시 해당 표준 분량이 집필 프롬프트에 자동 적용됩니다.

## 회차 제목 엔진 (v2.1 신규)

플랫폼 목록에서 독자가 다음 회차를 결제할지 결정하는 핵심 훅. **7~15자 이내** 제목을 6가지 유형으로 분류 생성합니다.

| 유형 | 예시 |
|------|------|
| 상황 제시형 | "벼랑 끝의 구원자" |
| 대사 인용형 | "그만 나오지?" |
| 미스터리 제기형 | "가면 뒤의 남자" |
| 감정 선언형 | "심장이 멈췄다" |
| 의외성형 | "죽었다던 사람" |
| 시공간형 | "세 번째 밤" |

**금지 사항:** 스포일러 노출, 단순 번호, 감정 형용사 나열, 장황한 설명.

## 웹소설식 줄바꿈 규칙 (v2.1 강화)

한국 웹소설 실전 양식 반영:

1. **문장 단위 줄바꿈** — 마침표 끝나면 엔터
2. **문단(덩어리) 2~4줄** — 문단 사이 빈 줄 하나
3. **벽돌 금지** — 여러 문장이 한 덩어리로 붙으면 탈락
4. **대사-지문 분리** — 영어권식 "said" 뒤붙이기 금지
5. **모바일 한 화면 3~5줄** — 스크롤 가독성 최우선

## 핵심 시스템

### 🎯 Core Arc + Extension 구조

```
[Core Arc 50화]        독립 완결 가능
  ↓ 반응 지표 확인
[Extension +25~50화]   인기 대응 확장 (3모드)
  ↓
[Season 2]             장기 IP화
```

**Extension 3가지 모드:**
- **Bridge Extension** (중반 30~35화) — 기존 회차 유지
- **Late Extension** (후반 45화) — 클라이맥스 유예
- **Season Break** (완결 후) — 독립 시즌 2

### 🏷️ 웹소설 공식 태그 9종

환생 · 회귀 · 빙의 · 역하렘 · 치정 · 계약결혼 · 먼치킨 · 복수 · 사이다

### 🔥 클리프행어 7유형

Slap · Reveal · Reversal · Arrival · Choice · Threat · Tears (3회차 연속 방지)

### 🌱 떡밥 관리 5종

핵심 · 중기 · 단기 · 미끼(Red Herring) · Extension예비

### 🔞 듀얼 수위

19금 원본(Opus) → 15금 변환(Sonnet). 서사적 기능 유지 변환.

## DOCX 출력 (v2.1 신규)

| 형식 | 용도 |
|------|------|
| **회차별 DOCX** | 개별 수정·편집 (제목+메타+본문+회차 메타) |
| **시즌 전체 DOCX** | 피칭/보관 (커버+목차+전체 본문) |
| **기획서 DOCX** | 투자 피칭 (컨셉+캐릭터+아크+떡밥) |
| **전체 ZIP** | 모든 회차 TXT+DOCX + 메타데이터 JSON |

블루진픽처스 디자인 시스템(네이비/옐로우, 노란 섹션 헤더, 한영 병기) 적용.
