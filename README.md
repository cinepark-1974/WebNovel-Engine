## v3.0의 4축

v3.0은 다음 4축의 모든 결정을 자동·반자동으로 도출합니다.

| 축 | 의미 |
|---|---|
| **장르** | 작품이 속한 시장 카테고리 + 6 포뮬러 + 23 모티프 + 5종 결합 패턴 |
| **캐릭터** | 외모·말투·행동 + 9종 인물 역할 + 모에 25속성 + 마음 흐름 5단계 아크 |
| **서사** | 회차별 플롯 + 6 포뮬러별 스토리 다이어그램 + 7종 클리프행어 분포 |
| **타겟** | 4종 페르소나 + 소비자 3분화(팬/매니아/오타쿠) + 결제 트리거 5종 + 시장 정량 가중치 |

   │          │              · 마음 흐름 5단계 자동 추론

## v3.0 신규 콘셉트 카드 필드

기존 v2.6.4 콘셉트 카드 위에 다음 필드가 자동 추가됩니다.

```json
{
  "formula_main": "운명적_인연물",
  "formula_sub": "갑을관계_파괴물",
  "movement_code": "빙의",
  "relationship_motifs": {
    "primary": "재회물",
    "secondary": ["운명적사랑", "갑을관계"],
    "tertiary": ["소유욕/독점욕/질투"]
  },
  "target_consumption_tier": ["매니아", "오타쿠"],
  "heroine_name": "한시호"
}
```

캐릭터 바이블에는 다음 필드가 추가됩니다.

```json
{
  "narrative_role": "전사_정신적_스승",
  "moe_attributes": ["역할:신비한_단골손님", "성격:쿨데레"]
}
```

이 필드들은 v3.0 5종 블록의 입력으로 사용됩니다. 비어있으면 v2.6.4와 동일하게 동작하므로 호환성 문제는 없습니다.

## v3.0 호환성

- v2.6.4 작품 데이터(JSON 콘셉트, 캐릭터 바이블) 그대로 사용 가능

- `use_v3_blocks=False`로 호출하면 v2.6.4와 완전 동일 동작

## 핵심 원칙

1. **기획↔집필 강제 연결** — 기획에서 정한 모든 재료가 회차 집필 프롬프트에 자동 주입
2. **시장 검증** — 5대 결정 요인(인기/소재/장르/평점/가격)이 회차마다 강제 충족
3. **캐릭터 평면화 방지** — 9종 인물 역할 자동 분배로 다중 인물 등장 회차에서 식별성 확보
4. **마음 흐름 일관성** — 회차별 단계 자동 추론으로 작품 전체의 정서적 흐름 유지
5. **여성향/남성향 발란스** — 양쪽 동등 지원 (마음 흐름 5단계 vs 완성된 영웅 3단계)
